A PROCEDURE IS A LOGICAL GROUP OF SQL AND PL/SQL STATEMENTS THAT PERFORM A SPECIFIC TASK. IT CAN ACCEPT PARAMETERS, PERFORM AN ACTION BUT CAN NVER RETURN A VALUE. PROCEDURES CAN BE OF 2 TYPES :
1) ANNONYMOUS PROCEDURES
2) STORED PROCEDURES

ANNOMYMOUS PROCEDURES: THIS TYPE OF PROCEDURE HAS NO NAME AND GETS EXECUTED AFTER SUBMITTING IT'S SOURCE CODE TO THE DATABASE SERVER BY USING @ OR RUN COMMAND AT SQL PROMPT. AN ANNONYMOUS PROCEDURE IS COMPILED EACH TIME IT'S CONTAINING BLOCK IS COMPILED. ALSO THESE PROCEDURES ARE NOT STORED AS A DATABASE OBJECT IN THE ORACLE DATABASE AND THAT IS WHY IT CANNOT BE CALLED FROM ANYWHERE ELSE. IT CANNOT ACCEPT PARAMETERS AND CANNOT RETURN VALUES AS WELL. IT MAY BE WRITTEN USING ANY STANDARD EDITOR MUST BE SAVED INTO A FILE WITH .SQL EXTENTION AND TO BE EXECUTED BY WRITING THE COMMAND @FILE_NAME_WITH_FULL_PATH OR RUN_FILE_NAME_WITH_FULL_PATH

STORED PROCEDURES : THIS TYPE OF PROCEDURES IS CREATED WITH A NAME AND IS STORED IN COMPILED FORMAT(KNOWN AS PCODE) AS AN OBJECT IN AN DATABASE. IT CAN ACCEPT INPUT PARAMETERS AND PAST VALUES TO OUTPUT PARAMETERES. BUT IT CANNOT RETURN VALUES TO IT'S CALLER. THE FLEXIBILITY OF THESE PROCEDURES IS THEY CAN BE INVOKED FROM VARIOUS ENVIRONMENTS THAT HAVE ACCESS TO THE DATABASE, FOR EXAMPLE FROM SQL PROMPT AS WELL AS FROM OTHER PROCEDURES OR FUNCTIONS. THE PL/SQL BLOCK OF STORED PROCEDURE IS SAVED IN A FILE WITH A .SQL EXTENTION. TO COMPILE A STORED PROCEDURE THE FOLLOWING COMMAND TO BE EXECUTED :
@FILE_NAME_WITH_FULL_PATH;

IF THE ORACLE ENGINE SUCCESFULLY COMPILES THE CODE THEN THE MESSAGE IS SHOWN "PROCEDURE CREATED", OTHERWISE AN ERROR MESSAGE COMES OUT AS "WARNING! PROCEDURE CREATED WITH COMPILATION ERROR". THE COMPILATION PROCESS DOES NOT DISPLAY THE ERRORS. ALL THE SYNTAX'S ARE ENUMERATED AS FOLLOWS :

LISTING OUT THE STORED PROCEDURES :
 SELECT OBJECT_NAME FROM USER_PROCEDURES

2. SYNTAX FOR CREATING STORED PROCEDURE :
CREATE OR REPALACE PROCEDURE PROCEDURE_NAME[(PARAMETER IN|OUT|IN OUT DATATYPE)] IS
	----DECLARATIVE SECTION
BEGIN
	----EXECUTION SECTION
EXCEPTION
	----EXCEPTION HANDLERS
END;
/

C) SYNTAX FOR COMPILING STORED PROCEDURE :

SQL> @FILE NAME WITH FULL PATH;
	- ON SUCCESFUL COMPILATION, THE MESSAGE SHOWN IS :
	--------------------------------------------------
	SQL> PROCEDURE CREATED
	- IF COMPILATION FAILS :
	_________________________________________
	SQL> WARNING : PROCEDURE CREATED WITH COMPILER ERROR
D) SYNTAX FOR SHOWING ERRORS IF COMPILATION FAILS :
---------------------------------------------------
SQL> SELECT TEXT FROM USER_ERRORS;

E) SYNTAX FOR EXECUTING SQL> PROMPT :
-------------------------------------
	1) EXECUTING FROM SQL> PROMPT
	-----------------------------
	SQL> EXEC PROCEDURE_NAME
		OR
	SQL> EXEC PROCEDURE_NAME(ARG1,ARG2,....);
	
	2) EXECUTING FROM ANOTHER PROCEDURE
	-----------------------------------
	BEGIN
		PROCEDURE_NAME;
			OR
		PROCEDURE_NAME(ARG1,ARG2,....);
		END;
		/
F) SYNTAX FOR DELETING THE PROCEDURE:
------------------------------------
SQL> DROP PROCEDURE PROCEDURE_NAME;

question : create a procedure sumproc with 2 numbers passed as parameters, to add the numbers passed and display the result.
answer :
if there is compilation error write select text from user_errors;
COMMAND PROMPT EXECUTION :
CLEAR SCREEN 
SET SERVEROUTPUT ON 

CREATE OR REPLACE PROCEDURE SUMPROC(N1 IN NUMBER, N2 IN NUMBER) IS
	V1 NUMBER;
	V2 NUMBER;
	V3 NUMBER;
BEGIN
	V1 := N1;
	V2 := N2;
	V3 := N1+N2;
	DBMS_OUTPUT.PUT_LINE('SUM OF '||V1||' AND '||V2||' = '||V3);
END;
/

POINTS TO BE REMEBERED :

1) IF NO PARAMETER IS TO BE PASSED TO THE PROCEDURE THEN NO PARENTHESIS SHOULD BE SPECIFIED WITH THE PROCEDURE NAME 
2) IF PARAMETER IS PASSED THEN THE SIZE CANNOT BE SPECIFIED FOR ANY DATATYPES NAMELY CHAR VARCHAR VARCHAR2  OR NUMBER IN THE SIGNATURE OF THE PROCEDURE 
3) USER SHOULD NOT BE ASKED FOR ENTERING VALUES BY USING ANY SUBSTITUTION VARIABLE IN ANY STORED PROCEDURE AS &EMPLOYEE_ID, BECAUSE THE USER WILL BE ASKED TO ENTER VALUES AT THE TIME OF COMPILATION AND WHATEVER THE VALUES ENTERED THAT BECOME PERMANENT IN THE COMPILED OBJECT AND NEVER CHANGED VALUES FURTHER WHEN THE PROCEDURE IS CALLED IN FUTURE. EVEN THE USER WILL NEVER BE ASKED TO ENTER THE VALUES. FOR EXAMPLE :

4) IN A PROCEDURE ANY PARAMETER CAN BE PASSED EITHER OF 3 MODES :
i) IN 
ii) OUT
iii) IN OUT
IF NOTHING IS EXPICITLY MENTIONED THEN BY DEFAULT IT IS TREATED AS IN MODE.

5) PARAMETER SPECIFIED IN A PROCEDURE CALL ARE KNOWN AS ACTUAL PARAMETERS, THOSE CAN BE LITERALS, NUMBERS , CONSTANTS , VARIABLES OR EXPRESSIONS. AND PARAMETERS SPECIFIED IN THE PARAMETER LIST OF A PROCEDURE DEFINATION ARE KNOWN AS FORMAL PARAMETERS. ON CALLING A PROCEDURE THE ACTUAL PARAMETER VALUES ARE ASSIGNED TO THE FORMAL PARAMETERS 

6) IN, OUT AND IN OUT ARE THE MODES IN WHICH PARAMETERS CAN BE PASSED INDICATING WHETHER PARAMETER IS BEING USED FOR INPUT, OUTPUT OR BOTH THE INPUT OUTPUT. IN IS THE DEFAULT PARAMTER MODE . THE FEATURES OF IN, OUT AND IN OUT ARE AS FOLLOWS :
 
IN	OUT	IN OUT
1) USED TO PASS ACTUAL PARAMETER VALUES TO FORMAL PARAMETERS	1) USED TO RETURN VALUES TO ACTUAL PARAMETERS FROM FORMAL PARAMETERS	1) USED TO PASS ACTUAL PARAMETERS AND RETURN UPDATED VALUES TO ACTUAL PARAMETERS
2) IT IS THE DEFAULT MODE, IF NO MODE IS EXPILICITLY SPECIFIED	2) IT MUST BE SPECIFIED EXPILCITLY	2)IT MUST BE SPECIFIED EXPILCITLY
3) THE ACTUAL VALUE IS PASSED TO THE FORMAL PARAMETER	3)THE NULL VALUE IS PASSED TO THE FORMAL PARAMETER	3) THE ACTUAL VALUE IS PASSED TO THE FORMAL PARAMETER.
4) THE VALUE OF FORMAL PARAMETER CANNOT BE CHANGED	4) THE VALUE OF FORMAL PARAMETER CAN BE CHANGED AND DOES REFLECT TO THE ACTUAL	4) THE VALUE OF FORMAL PARAMETER CAN BE CHANGED AND DOES REFLECT TO THE ACTUAL
5)CANNOT BE INITIALISED WITH ANY DEFAULT VALUE 	5)CANNOT BE INITAILSED WITH ANY DEFAULT VALUE.	5)CANNOT BE INITAILSED WITH ANY DEFAULT VALUE.

7) IN CASE OF OUT PARAMETER, AN ACTUAL PARAMETER MAY HAVE A VALUE BEFORE THE PROCEDURE IS CALLED HOWEVER THE VALUE IS LOST AND INITIALISED TO NULL WHEN THE PROCEDURE IS CALLED. THEREFORE IT'S VALUE CANNOT BE ASSIGNED TO ANOTHER VARIABLE OR REASSIGNED TO ITSELF. INSHORT AN OUT PARAMETER CANNOT APPEAR IN THE RIGHT HAND SIDE OF AN ASSIGNMENT OPERATOR(:=) AND BEFORE EXITING A PROCEDURE, SOME VALUES SHOULD EXPILCITLY BE ASSIGNED TO ALL THE OUT FORMAL PARAMETERS; OTHERWISE THE CORRESONDING ACTUAL PARAMETERS SHALL BE NULL, EXPLAIN WITH A FOLLOWING EXAMPLE.

LOCAL PROCEDURE OR NESTED PROCEDURE :

ANOTHER VARITION OF PROCEDURE IS LOCAL PROCEDURE WHICH IS NEITHER AN ANNONYMOUS PROCEDURE (AS IT HAS GOR A NAME) NOR A STORED PROCEDURE(AS IT IS NOT STORED AS DATABASE OBJECT IN ORACLE). THIS PROCEDURE IS DECLARED AND USED WITHIN A PROCEDURE(MAYBE EITHER ANNONYMOUS OR STORED PROCEDURE) AND RESTRICTED TO THE SCOPE OF THAT PROCEDURE ONLY WHERE IT IS DECLARED OR CREATED. THIS PROCEDURE IS CAPABLE OF HANDLING PARAMETERS PASSED BY THE CALLING SEGMENT. BUT THE IMPORTANT THING IS THIS KIND OF PROCEDURES CANNOT BE CREATED WITH THE TEXT PHRASE " CREATE OR REPLACE",BUT THE REST OF THE DECLARATION IS SAME AS THAT OF STORED PROCEDURE. 

LOCAL PROCEDURE CREATED WITHIN ANNONYMOUS PROCEDURE :

WRITE A LOCAL PROCEDURE THAT CALCULATES SUM OF TWO PARAMETERS GIVEN BY USER 

CLEAR SCREEN
SET SERVEROUTPUT ON
DECLARE
	A NUMBER;
	B NUMBER;
	S NUMBER;
	PROCEDURE SUMMATION(X IN NUMBER, Y IN NUMBER, Y IN NUMBER, Z OUT NUMBER) IS
	BEGIN
		Z := X+Y;
	END;
BEGIN
	A := &NUMBER1;
	B := &NUMBER2;
	SUMMATION(A,B,S);
	DBMS_OUTPUT.PUT_LINE('SUM OF ||A||'AND'||B||'='||S);
END;
/

LOCAL PROCEDURE CREATED WITHIN STORED PROCEDURE :

WRITE A LOCAL PROCEDURE THAT CALCULATES FACTORIAL OF A NUMBER PASSED AS PARAMETER, FOR CALCULATING 1!+3!+5!+...UPTO TEN

CLEAR SCREEN
SET SERVEROUTPUT ON

CREATE OR REPLACE PROCEDURE FACT_SERIES IS 
	N NUMBER;
	F NUMBER;
	S NUMBER;
	PROCEDURE FACT(X IN NUMBER, P OUT NUMBER) IS
	BEGIN
		P:=1;
		FOR I IN 1..X LOOP
			P:= P*I;
		END LOOP;
	END;
BEGIN
	N:=1;
	F:=0;
	S:=0;
	WHILE N<=10 LOOP
		FACT(N,F);
		S:= S+F;
		N:= N+2;
	END LOOP;
	DBMS_OUTPUT.PUT_LINE('SUM ='||S);
END;
/
