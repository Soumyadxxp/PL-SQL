EXPLICIT CURSOR :
_________________
EXPLICIR CURSORS ARE DELARED BY THE USER AND ARE USED TO PROCESS QUERY RESULTS THAT RETURN MULTIPLE ROW.
WHEN INDIVIDUAL RECORDS IN A TABLE HAD TO BE PROCESSED INSIDE A PL/SQL BLOCKTHIS TYPE OF CURSOR IS USED. THIS CURSOR HAD TO BE DECLARED AND MAPPED TO AN SQL QUERIE IN THE "DECLARED" SECTION OF THE PL/SQL BLOCK AND USED WITHIN IT'S EXECUTABLE SECTION. THE DATA THAT IS STORED IN THE CURSOR IS CALLED "ACTIVE DATASET. TO USE EXPLICIT CURSOR IN PL/SQL BLOCK, THE FOLLOWING 4 STEPS NEED TO BE GONE THROUGH :

1) DECLARING CURSOR :- IN THIS PROCESSS THE CURSOR IS DEFINED THAT IS , IT IS ASSIGNED A NAME AND GETS ASSOCIATED WITH A SELECT STATEMENT, WHICH MAY RETURN A SINGLE OR MULTIPLE ROW RECORD SET OR AN EMPTY SET.
TO BE REMEMBERED THAT IF, IN ANY CASE, THE SELECT STATEMENT DOESNOT SUCCED AND NO DATA IS FOUND, IT NEVER RAISES THE EXCEPTION NO_DATA_FOUND , RATHER RETURNS AN EMPTY RECORD SET. THE SYNTAX FOR DECLARING A CURSOR IS :

SYNTAX :

CURSOR CURSOR_NAME IS SELECT STATEMENT; 

EXAMPLE : CURSOR MYCURSOR IS SELECT FIRST_NAME FROM EMPLOYEES WHERE DEPARTMENT_ID = 30;



2) OPENING CURSOR :- IN THIS STEP THE CURSOR IS OPPENED THAT IS ORACLE EXECUTES THE SELECT STATEMENT ASSOCIATED WITH THE CURSOR AND SETS THE ROW POINTER TO THE FIRST RECORD. IF ANY ATTEMPT IS MADE TO OPEN A CURSOR ALREADY OPPENED THEN PL/SQL RAISES THE EXCEPTION HANDLER " CURSOR_ALREADY_OPEN"
THE SYNTAX FOR OPENING A CURSOR IS :
SYNTAX : OPEN CURSOR_NAME;
EXAMPLE: OPEN MYCURSOR;

3)FETCHING ROWS FROM CURSOR :- IN THIS STEP ONE ROW IS RETRIVED FROM FROM ACTIVE SET, ASSIGNS DATA INTO THE CORRESPONDING PL/SQL VARIABLES AND RESETS THE ROW POINTER TO MOVE FORWARD TO THE NEXT RECORD. THE SYTANX FOR FETCHING A RECORD FROM THE ACTIVE SET IS:
SYNTAX  : FETCH CURSOR_NAME INTO RECORD_LIST;

EXAMPLE : DECLARE
             ENAME EMPLOYEES.FIRST_NAME%TYPE;
        BEGIN
            FETCH MYCURSOR INTO ENAME;
            DBMS_OUTPUT.PUT_LINE(ENAME);
        END;

4) CLOSING THE CURSOR : IN IT'S FINAL STEP THE CURSOR IS CLOSED WHICH MEANS THAT ALL RESOURCES CONSUME BY ORACLE RELATED TO THE CURSOR ARE RELEASED. 
SYNTAX : CLOSE CURSOR_NAME;
EXAMPLE : CLOSE MYCURSOR; 

EXPLICIT CURSOR ATTRIBUTES :

ATTRIBUTE	DESCRIPTION
%ISOPEN	EVALUATES TO TRUE IF AN EXPILICIT CURSOR OIS OPPENED OTHERWISE TO FALSE. SYNTAX : CURSOR_NAME%ISOPEN
%FOUND	EVALUATES TO TRUE IF THE LAST FETCHE SUCCEDS OTHERWISE TO FALSE. SYNTAX : CURSOR_NAME%FOUND
%NOTFOUND	EVALUATES TO TRUE IF THE LAST FETCHED FAILS OTHERWISE TO FALSE.
SYNTAX :
 CURSOR_NAME%NOTFOUND
%ROWCOUNT	RETURNS THE NUMBER OF ROWS FETCHED FROM THE ACTIVE SET. IT IS SET TO 0 WHEN THE CURSOR IS OPPENED. THEREAFTER IT RETURNS THE NO. OF ROWS FETCHED SO FAR. THE NO. ID INCREMENTED BY 1 IF THE LATEST FETCH RETURNS A ROW. 
SYNTAX : 
CURSOR_NAME%ROWCOUNT

EXAMPLE QUESTION > ACCEPT A DEPARTMENT NUMBER AND DISPLAY THE DETAILS OF ALL THE EMPLOYEES WORKING IN THE DEPARTMENT ALONG WITH THE TOTAL NUMBER OF EMPLOYEES. IF NONE IS WORKING IN THE DEPARTMENT THEN DISPLAY THE TOTAL NUMBER OF EMPLOYEES = 0;

ANSWER >

SET SERVEROUTPUT ON
DECLARE
	CURSOR EMPCURSOR IS SELECT * FROM EMPLOYEES WHERE DEPARTMENT_ID = &DEPARTMENT_NUMBER;
	REC EMPLOYEES%ROWTYPE;
BEGIN
	OPEN EMPCURSOR;
	FETCH EMPCURSOR INTO REC;
	IF EMPCURSOR%NOTFOUND THEN
		DBMS_OUTPUT.PUT_LINE('NUMBER OF EMPLOYEES = 0');
	ELSE
		DBMS_OUTPUT.PUT_LINE('---+----------+---');
		DBMS_OUTPUT.PUT_LINE('ENO|FIRST NAME|DNO');
		DBMS_OUTPUT.PUT_LINE('---+----------+---');
		LOOP
			DBMS_OUTPUT.PUT_LINE(REC.EMPLOYEE_ID||'|'||RPAD(REC.FIRST_NAME,10)||'|'||LPAD(REC.DEPARTMENT_ID,3));
			FETCH EMPCURSOR INTO REC;
			EXIT WHEN EMPCURSOR%NOTFOUND;
		END LOOP;
		DBMS_OUTPUT.PUT_LINE('---+----------+---');
		DBMS_OUTPUT.PUT_LINE('TOTAL EMPLOYEES:'||EMPCURSOR%ROWCOUNT);
		DBMS_OUTPUT.PUT_LINE('---+----------+---');
	END IF;
END;

PARAMETERISED CURSOR : THE CURSORS WITH THE ACTIVE DATASETS BASED ON SOME CONSTANT CRITERIA SPECIFIED IN THE WHERE CLAUSE OF THE SELECT STATEMENT ARE HARD CODED AND NEVER CHANGES. COMMERCIAL APPLICATIONS REQUIRE THAT THE QUERIE, WHICH DEFINES THE CURSOR, BE GENERIC AND THE DATA THAT IS RETRIVED FROM THE TABLE BE ALLOWED TO BE CHANGED ACCORDING TO THE NEED. 

ORACLE PERMITS  THE CREATION OF CURSOR WITH SOME VALUES PASSED AS PARAMETERS INTO THEM. SINCE THE CURSOR ACCEPT USER DEFINED VALUES INTO THEIR PARAMETERS, THUS CHANGE THE RESULT SETS EXTRACTED, THEY ARE CALLED PARAMETERISED CURSOR. 

THE SYNTAX FOR DECLARING PARAMETERISED CURSOR : 

CURSOR CURSOR_NAME (PARA1 DATATYPE [DEFAULT DEFAULT_VALUE],....) IS SELECT STATEMENT;

THE SYNTAX FOR OPENING AND PASSING VALUES TO THE CURSOR :

OPEN CURSOR_NAME('STRING LITERAL'|NUMBER|VARIABLE|EXPRESSION);

Q8) CLEAR SCREEN
SET SERVEROUTPUT ON

DECLARE
    V_DEPT_NAME DEPARTMENTS.DEPARTMENT_NAME%TYPE := '&DEPARTMENT_NAME';

    CURSOR EMPCURSOR IS 
        SELECT E.FIRST_NAME, E.JOB_ID, E.SALARY
        FROM EMPLOYEES E, DEPARTMENTS D
        WHERE E.DEPARTMENT_ID = D.DEPARTMENT_ID
        AND D.DEPARTMENT_NAME = V_DEPT_NAME
        ORDER BY E.JOB_ID;

    TYPE EMPRECORD IS RECORD 
    ( 
        FNAME EMPLOYEES.FIRST_NAME%TYPE,
        JOB   EMPLOYEES.JOB_ID%TYPE,
        SAL   EMPLOYEES.SALARY%TYPE
    );

    REC EMPRECORD;

    JOBTOTSAL NUMBER := 0;
    JOBHEAD NUMBER := 0;
    GRANDTOTSAL NUMBER := 0;

    PREVJOB EMPLOYEES.JOB_ID%TYPE;

BEGIN
    OPEN EMPCURSOR;
    FETCH EMPCURSOR INTO REC;
    PREVJOB := REC.JOB;

    DBMS_OUTPUT.PUT_LINE('NAME      | JOB ID    | SALARY');

    WHILE EMPCURSOR%FOUND LOOP

        DBMS_OUTPUT.PUT_LINE(RPAD(REC.FNAME,10)||'|'||RPAD(REC.JOB,10)||'|'||LPAD(REC.SAL,6));

        JOBTOTSAL := JOBTOTSAL + REC.SAL;
        GRANDTOTSAL := GRANDTOTSAL + REC.SAL;
        JOBHEAD := JOBHEAD + 1;

        FETCH EMPCURSOR INTO REC;

        IF PREVJOB <> REC.JOB THEN
            DBMS_OUTPUT.PUT_LINE('------------------------------');
            DBMS_OUTPUT.PUT_LINE('SALARY FOR '||JOBHEAD||' HEADS: '||JOBTOTSAL);
            DBMS_OUTPUT.PUT_LINE('------------------------------');
            JOBTOTSAL := 0;
            JOBHEAD := 0;
        END IF;

        PREVJOB := REC.JOB;

    END LOOP;

    DBMS_OUTPUT.PUT_LINE('------------------------------');
    DBMS_OUTPUT.PUT_LINE('SALARY FOR '||JOBHEAD||' HEADS: '||JOBTOTSAL);
    DBMS_OUTPUT.PUT_LINE('------------------------------');
    DBMS_OUTPUT.PUT_LINE('GRAND TOTAL SALARY : '||GRANDTOTSAL);

END;

CURSOR FOR LOOP :

A SPECIAL PURPOSE OF USING FOR LOOP IS MANIPULATION OF RECORDS IN A CURSOR. IT IS A MACHINE DEFINED LOOP EXIT i.e. when all the values in the FOR construct are exhausted, the looping stops automatically.

SYNTAX :
	FOR FOR_VARIABLE IN CURSOR_NAME LOOP
		....
	END LOOP;
HERE THE FOR AUTOMATICALLY CREATES THE FOR VARIABLE OF %ROW TYPE. EACH RECORD IN THE OPEN CURSOR BECOME A VALUE FOR THE FOR VARIABLE OF THE %ROWTYPE. 
THE FOR VERB ENSURES THAT A ROW FROM THE CURSOR IS LOADED IN THE DECLARED FOR VARIABLE AND THE LOOP EXECUTE ONCE. THIS GOES ON UNTILL ALL THE ROWS OF THE CURSOR HAVE BEEN LOADED INTO THE FOR VARIABLE AFTER THIS THE LOOP STOPS.

THE FOR LOOP AUTOMATICALLY DOES THE FOLLOWINGS :

IMPLICITLY DECLARES IT'S LOOP INDEX AS A %ROWTYPE RECORD
OPEN'S A CURSOR
FETCH'S A ROW FROM THE CURSOR FOR EACH LOOP ITERATION
CLOSES THE CURSOR WHEN ALL THE ROWS HAVE BEEN PROCESSED

A CURSOR CAN BE CLOSED EVEN WHEN AN EXIT OR GOTO STATEMENT IS USED TO LEAVE THE LOOP PREMATURELY OR AN EXCEPTION IS RAISED INSIDE THE LOOP.

EXAMPLE :

Q) DISPLAY FIRST_NAME, JOB_ID AND DEPARTMENT_ID OF ALL THE EMPLOYEES WHO ARE WORKING IN SEATTLE. ALSO DISPLAY HOW ANY EMPLOYEES ARE WORKING IN THE CITY.

ANS)

CLEAR SCREEN
SET SERVEROUTPUT ON
DECLARE 
	CURSOR MYCUR IS SELECT FIRST_NAME,JOB_ID,DEPARTMENT_ID FROM EMPLOYEES WHERE DEPARTMENT_ID =ANY (SELECT DEPARTMENT_ID FROM DEPARTMENTS WHERE LOCATION_ID = (SELECT LOCATION_ID FROM LOCATIONS WHERE CITY = 'Seattle')) ORDER BY DEPARTMENT_ID;
	COUNTER NUMBER(2):=0;
BEGIN
	DBMS_OUTPUT.PUT_LINE('----------+--------+-------');
	DBMS_OUTPUT.PUT_LINE('FIRST NAME|JOB ID  |DEPT ID');
	DBMS_OUTPUT.PUT_LINE('----------+--------+-------');
	FOR REC IN MYCUR LOOP
		DBMS_OUTPUT.PUT_LINE(RPAD(REC.FIRST_NAME,10)||'|'||RPAD(REC.JOB_ID,8)||'|'||LPAD(REC.DEPARTMENT_ID,7));
		COUNTER := COUNTER + 1;
	END LOOP;
	DBMS_OUTPUT.PUT_LINE('----------+--------+-------');
	DBMS_OUTPUT.PUT_LINE('NUMBER OF EMPLOYEES : '||COUNTER);
	DBMS_OUTPUT.PUT_LINE('----------+--------+-------');
END;
/
